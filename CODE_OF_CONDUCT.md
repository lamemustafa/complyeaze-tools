# Code of Conduct

ComplyEaze Tools project spaces should stay respectful, specific, and
evidence-led.

## Expected Behavior

- Use synthetic examples when discussing bugs, fixtures, or screenshots.
- Keep disagreements focused on source evidence, product boundaries, security,
  privacy, accessibility, and maintainability.
- Respect contributor time by including reproduction steps and verification
  results when possible.

## Unacceptable Behavior

- Posting taxpayer, client, employee, vendor, portal, credential, bank, OTP,
  CAPTCHA, cookie, document, notice, ledger, or screenshot data in public.
- Harassment, threats, discriminatory language, or personal attacks.
- Pressuring maintainers to accept claims or tool behavior without source
  support.

## Enforcement

Report conduct concerns to contact@complyeaze.com. Security and privacy issues
must go to security@complyeaze.com instead of public project spaces.

Maintainers may edit or remove public content that exposes sensitive data,
close issues that cannot be safely discussed publicly, or block participants who
repeatedly violate this policy.
